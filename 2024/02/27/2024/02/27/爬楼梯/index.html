leetcode70：爬楼梯

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

解题思路：

  1. 一共要爬 n 阶，n >= 0；
  2. 一次可以爬 1 或 2 个台阶；
  3. 问爬 n 阶有多少种爬法（重点加粗了，问的是爬法，走是不算的）；

假设$f(n)$表示爬n阶楼梯所需要的跳法。

倒推上一步的可能情况：

***$f(n)=f(n-1)+f(n-2)$***，考虑最简单的做法就是无脑递归。

# 思路是递归每一级台阶可能的跳法，当位于第1或第0级台阶只有一种跳法，并返回。
class Solution:
    def climbStairs(self, n: int) -> int:
        if n == 0 or n == 1:
            return 1

        return self.climbStairs(n - 1) + self.climbStairs(n - 2)

这种方法可行，但是对于有时间要求的题目，会无法通过，这是因为我们没有考虑到重叠子问题和最优子结构特点的问题，造成了不必要的递归开销。对于每一级台阶，到达它的方法不会变化，但是到达它的更高处台阶是不同的，这就会造成每一级台阶的反复递归计算。

于是我们想到使用记忆化递归。

class Solution:
    def climbStairs(self, n: int) -> int:
        
        def digui(i, memory):
            if i == 0 or i == 1:
                return 1
            
            if memory[i] == -1:
                memory[i] = digui(i-1, memory) + digui(i-2, memory)
            return memory[i]
            
        return digui(n, [-1]*(n + 1) )

这种方式成功解决了重叠子问题不必要的递归开销。当然，动态规划也可以解决此类问题，让我们往下看。

class Solution:
    def climbStairs(self, n: int) -> int:
        
        dp = [-1]*(n + 1)
        dp[0] = dp[1] = 1
        for i in range(2,n+1):
            dp[i] = dp[i-1] + dp[i-2]
        
        return dp[-1]

很像，很像，是的，很像斐波那契数列，这个问题就是一种类斐波那契数列问题，只是刚好最多挑两级台阶，所以和斐波那契数列一样。

既然是斐波那契数列，那我们继续优化。

class Solution:
    def climbStairs(self, n: int) -> int:
        a = b = 1
        for i in range(2, n + 1):
            a, b = b, a+b
        return b

总结

动态规划（DP）和记忆化递归是解决具有重叠子问题和最优子结构特点的问题的两种主要方法。它们都旨在减少重复计算，但它们的实现方式和概念框架有所不同。

记忆化递归 (Memoization) 概念：记忆化递归是一种自顶向下的方法，它是递归的优化版本。在递归调用过程中，它会保存计算过的结果（通常是在数组或哈希表中），以防止对相同输入的重复计算。

实现方式：通常在递归函数中，一旦计算出一个结果，就会将其存储在数据结构中（例如，数组或哈希表）。在递归调用之前，会首先检查是否已经计算过当前的输入，如果是，则直接返回存储的结果。

特点：记忆化递归通常更容易实现和理解，因为它紧跟问题的递归结构。它的缺点是可能会遇到递归调用的栈溢出问题，特别是在深度递归的情况下。

动态规划 (DP) 概念：动态规划是一种自底向上的方法。它通过将问题分解成较小的子问题，然后解决这些子问题，并将解决方案组合起来解决原始问题。

实现方式：在动态规划中，通常会先计算所有较小的子问题，然后逐步构建出解决原始问题的答案。这通常在一个表格中进行，每个表格项代表一个子问题的解。

特点：动态规划通常更高效，因为它避免了递归调用的开销，减少了栈空间的使用。它的缺点是实现起来可能比记忆化递归更复杂，特别是当问题的状态空间很大或者状态转移不是很明显时。

总结 记忆化递归：更接近于问题的自然递归形式，实现相对简单，但可能导致栈溢出。 动态规划：通常更高效，避免了递归开销，但可能在实现上更复杂，特别是在状态转移不直观的情况下。 在实践中，选择哪种方法取决于具体问题的性质和个人对问题的理解。有时，两种方法可以交替使用或结合使用，以达到最佳的解决方案。
